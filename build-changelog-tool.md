# BUILD — Changelog to launch post

**Input:**

> I want to build a tool for indie developers that turns their changelog into a
> tweet-sized launch post. Users paste a changelog, pick a tone, and get 3
> candidate posts.

Scores: 5 / 4 / 4. Zero external data, zero safety risk, three features.

---

## 1. Verdict

BUILD

## 2. Target User

Indie developers looking to streamline their social media marketing efforts by
easily converting changelogs into engaging posts.

## 3. Core Problem

The challenge of effectively promoting updates and new features on social media
in a concise format.

## 4. Evaluation Scores

- **User Clarity Score:** 4 — Indie developers are a specific group with a clear
  need for social media marketing tools.
- **Problem Strength Score:** 4 — Promoting updates on social media is a clear
  pain point, as developers often struggle to effectively communicate changes and
  features.
- **MVP Feasibility Score:** 4 — The requested features are straightforward and
  can be implemented within a week by a beginner, focusing on text processing and
  basic tone selection.

## 5. Why It Might Work

Indie developers often seek efficient ways to engage their audience. This tool
simplifies the process of creating launch posts, allowing them to focus on
development instead of marketing.

## 6. Main Risks

- The tool may not capture the unique voice of each developer.
- Users may prefer more customization options than the MVP provides.
- Market competition could limit user adoption.

## 7. Revised Idea

A tool that allows indie developers to input a changelog, select a tone, and
receive three tweet-sized candidate posts to promote their updates effectively.

## 8. MVP Features

1. **Changelog Input:** Users can paste their changelog into a text box.
2. **Tone Selection:** Users can choose from predefined tones (e.g. professional,
   casual, enthusiastic).
3. **Post Generation:** The tool generates three tweet-sized posts based on the
   input changelog and selected tone.

## 9. 7-Day Building Plan

- **Day 1:** Research and outline the basic structure of the tool. Decide on
  programming language and framework.
- **Day 2:** Set up the development environment and create the basic UI for input
  and tone selection.
- **Day 3:** Implement the changelog input feature and ensure it captures user
  input correctly.
- **Day 4:** Develop the tone selection functionality with predefined options.
- **Day 5:** Create the algorithm to generate three candidate posts based on the
  input changelog and tone.
- **Day 6:** Test the tool for functionality and ensure it generates posts
  correctly.
- **Day 7:** Polish the UI and prepare for initial user feedback.

## 10. Future Version

- Integrate real-time data for trending topics to suggest relevant tones.
- Allow users to customize the generated posts further.
- Implement sharing features directly to social media platforms.

## 11. X/Twitter Launch Post

Just built a tool for indie devs to turn changelogs into tweet-sized launch
posts. Paste your changelog, pick a tone, and get 3 candidate tweets!
